# Plan 41-07: PII-Scrubbing für Sentry + Logs (TOM-FIX-04 + TOM-FIX-05)

## Goal

Verhindere PII-Leakage über zwei Telemetrie-Kanäle: (a) Sentry-Events (Frontend + Backend), (b) Server-Logs (Pino) und Frontend-Console. Zentrale Scrubbing-Util `apps/web/server/utils/scrub.ts` (auch im Frontend wiederverwendbar via re-export).

## Scrubbing-Regeln (zentralisiert in `scrub.ts`)

**Felder, die immer redaktiert werden:**
- `password`, `passwordHash`
- `verificationToken`, `verificationTokenHash`
- `resetToken`, `resetTokenHash`
- `googleIdHash`, `githubIdHash`
- `stripeCustomerId`, `stripeSubscriptionId`
- `apiKey`, `keyHash`
- `Authorization` Header → `Bearer [redacted]`
- `Cookie` Header → `[redacted]`

**Felder, die in URL Query-Params redaktiert werden:**
- `?token=`, `?verification_token=`, `?reset_token=`, `?code=`, `?state=`

**E-Mail-Pattern in Stack-Traces:**
- `xxx@yyy.zz` → `[redacted-email]`

## Steps

1. `apps/web/server/utils/scrub.ts` mit `scrubObject(obj)` + `scrubUrl(url)` + `scrubString(str)` Funktionen
2. Unit-Tests in `scrub.test.ts` decken alle Felder + URL-Patterns + Stack-Trace-Email
3. **Backend Sentry**: `apps/web/server/index.ts` ergänzt `beforeSend(event)`:
   ```typescript
   Sentry.init({
     ...,
     beforeSend(event) {
       if (event.user?.email) event.user.email = '[redacted-email]';
       if (event.request?.url) event.request.url = scrubUrl(event.request.url);
       if (event.request?.headers) event.request.headers = scrubObject(event.request.headers);
       if (event.request?.data) event.request.data = scrubObject(event.request.data);
       return event;
     },
   });
   ```
4. **Frontend Sentry**: `apps/web/src/main.tsx` analog (gleicher `beforeSend`)
5. **Pino-Logger**: `server/utils/logger.ts` ergänzt `redact: { paths: [...], censor: '[redacted]' }` (Pino-eigene Redaction) UND wrappt `pino()` mit Custom-Formatter, der URLs in `req.url`/`msg` scrubbt
6. **Frontend Console-Wrapper**: `src/lib/logger.ts` ersetzt `console.log/warn/error` Aufrufe in der Codebase (typed wrapper) und scrubbt vor Ausgabe
7. Code-Migration: einmaliges Codemod / sed um `console.log(` → `logger.log(` (NUR App-Code, nicht `node_modules`/`dist`)

## Verification

- `pnpm test:run --filter scrub` grün
- E2E-Smoke: Auth-Flow durchführen → Sentry-Test-Event → Sentry-Issue zeigt `[redacted-email]` statt echter E-Mail
- Backend-Logs: `tail logs/*.log` während eines Reset-Flows → kein E-Mail-Klartext, kein Token-Klartext
- Static check: `grep -rn "console\." apps/web/src/ apps/web/server/` zeigt nur `console`-Zugriffe in Logger-Wrapper-Files
