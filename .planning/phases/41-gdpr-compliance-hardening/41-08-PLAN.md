# Plan 41-08: Age-Gate + T&C-Acknowledgement (TOM-FIX-06)

## Goal

DSGVO Art. 8 (Mindestalter 16 für Einwilligung in Online-Dienste in Deutschland) und nachweisbare Akzeptanz der AGB + Datenschutzerklärung bei Registrierung. Beide werden mit `tcAcceptedVersion` versioniert, damit künftige AGB-Updates dokumentierte Re-Akzeptanz auslösen können.

## Schema-Migration

```prisma
model User {
  // ... existing fields ...
  birthYear         Int?      // nullable für Bestands-Accounts; pflicht bei neuen Registrierungen
  tcAcceptedAt      DateTime? // Timestamp der ersten T&C-Akzeptanz
  tcAcceptedVersion String?   // z.B. "1.0.0"
}
```

Bestands-Accounts: `birthYear` und `tcAccepted*` bleiben null → nicht-blockierend für Login, aber Banner bei nächstem Login mit Aufforderung zur Nachholung (separater Folge-Plan / nicht hier).

## Frontend (Register.tsx)

Neue Form-Felder:
1. **Geburtsjahr** (number-Input, `min={currentYear - 120}`, `max={currentYear - 16}`, required) — Validierung clientseitig, Submit blockiert bei `< 16`
2. **T&C-Checkbox** — required Checkbox: "Ich habe die [Datenschutzerklärung](/privacy) und die [AGB](/terms) gelesen und akzeptiert" (Links öffnen jeweilige Seite in neuem Tab)
3. Submit-Button disabled bis beide Felder valid

## Backend (auth.ts register handler)

Zod-Schema erweitern:
```typescript
const RegisterSchema = z.object({
  email: z.string().email(),
  password: z.string().min(12),
  name: z.string().min(1),
  birthYear: z.number().int().min(currentYear - 120).max(currentYear - 16),
  tcAccepted: z.literal(true),  // muss explizit true sein
  tcAcceptedVersion: z.string(),
});
```

Bei `currentYear - birthYear < 16`: 400 mit klarer Fehlermeldung "Mindestalter 16 Jahre. Bei jüngeren Nutzern ist Eltern-Einwilligung erforderlich — bitte privacy@newshub.app kontaktieren." Backend setzt `birthYear`, `tcAcceptedAt = new Date()`, `tcAcceptedVersion = body.tcAcceptedVersion`.

## AGB-Seite (`/terms`)

Neue `Terms.tsx` mit minimalen AGB:
- Vertragsgegenstand
- Nutzungsregeln
- Haftungsausschluss
- Beendigung
- Anwendbares Recht (DE)
- Verlinkung zur Privacy-Seite

i18n-Namespace `terms` analog zu `privacy`. **Anwalts-Review vor Produktion empfehlenswert** — initial Draft-Status in `docs/legal/TERMS-VERSIONS.md` notieren.
