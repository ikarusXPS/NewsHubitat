## Test Pattern Decisions

### Dynamic mock state via mutable module-level variables
```typescript
let mockTeams: Array<...> = [];
vi.mock('../hooks/useTeams', () => ({
  useTeams: () => ({ teams: mockTeams, ... }),
}));
// In each test: mockTeams = [...desired state...] before render
```
Vitest hoists `vi.mock` calls to the top of the file. Mutable `let` variables are captured by the factory closure and can be mutated in each test, which is the correct way to achieve dynamic hook returns without re-defining the mock per test.

### confirm() stubbing
```typescript
vi.stubGlobal('confirm', () => true);
// ... in beforeEach:
vi.unstubAllGlobals();
```
This is Vitest's idiomatic approach (vs `window.confirm = jest.fn()`). `vi.unstubAllGlobals()` ensures clean state per test.

### createPortal testing
BookmarkButton renders its dropdown via `createPortal(..., document.body)`. React Testing Library's `screen.getByText()` searches the full document including portals, so no special setup is needed. `container.querySelector` only searches within the component root — portal elements must be found via `screen.*` or `document.body.querySelector`.

### framer-motion mock
`motion.div` replaced with a plain `div` pass-through to prevent animation timer issues in test environment. `AnimatePresence` replaced with a fragment.
