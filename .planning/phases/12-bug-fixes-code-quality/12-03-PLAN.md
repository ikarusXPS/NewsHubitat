2. Run TypeScript with strict unused checks:

<task type="auto">
  <name>Task 2: Analyze unused exports</name>
  <files>src/**, server/**</files>
  <read_first>
    - D:\NewsHub\src\lib (utility libraries)
    - D:\NewsHub\server\services (backend services)
  </read_first>
  <action>
1. Use grep to find exports and check for imports:
```bash
# Find all exported functions
grep -r "export function\|export const\|export class" src/ server/ --include="*.ts" --include="*.tsx" | head -30
```

2. For each export, verify it's imported somewhere:
```bash
# Check if function is imported
grep -r "import.*{.*functionName" src/ server/ --include="*.ts" --include="*.tsx"
```

3. Focus on:
   - src/lib/ utilities
   - server/services/ service methods
   - src/hooks/ custom hooks

4. Remove exports that are:
   - Never imported anywhere
   - Not used in the same file
   - Not part of public API
  </action>
  <verify>
    <automated>echo "Manual verification needed for dead code removal"</automated>
  </verify>
  <acceptance_criteria>
    - Unused exports identified
    - Safe-to-remove items documented
    - No breaking changes to imports
  </acceptance_criteria>
  <done>Unused exports analyzed</done>
</task>

<task type="auto">
  <name>Task 3: Remove confirmed dead code</name>
  <files>Files with dead code</files>
  <read_first>
    - Analysis from Tasks 1-2
  </read_first>
  <action>
For each confirmed dead code item:

1. Verify it's truly unused (not dynamically imported, not in tests)
2. Remove the code
3. Run build and tests after each removal to ensure no breakage

Safety checks before removal:

- Not called dynamically (string-based lookups)
- Not part of a public interface
- Tests don't rely on it
  </action>
