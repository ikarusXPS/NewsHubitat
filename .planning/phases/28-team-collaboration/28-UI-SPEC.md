## Visual Focal Point

**Primary screen:** `/team/:id` (TeamDashboard)

**Focal point:** Team bookmark list positioned in the main content area (center/left on desktop, full-width on mobile). The bookmark list is the primary visual anchor because:
1. It displays the team's shared content — the core purpose of the feature
2. It occupies the largest visual area (60%+ of viewport)
3. Real-time bookmark updates draw attention here via slide-in animation

**Secondary anchor:** Team header with team name, member count badge, and role indicator. Positioned at top of content area, provides team context before scrolling to bookmarks.

**Visual hierarchy:**
1. Team header (name + member count) — establishes context
2. Bookmark list — primary interaction zone
3. Member sidebar (desktop) or collapsible member section (mobile) — supporting information

## Interaction Patterns

### Team Switcher Dropdown

- **Trigger:** Click on current team name/icon in header
- **Content:** List of user's teams (max 10), divider, "Create Team" button
- **Active state:** Cyan left border + background glow (per existing sidebar-link.active pattern)
- **Mobile:** Full-width dropdown in MobileDrawer

### Save to Team Flow

1. User clicks bookmark button on article
2. If user has teams: dropdown appears with "Personal" and team list
3. User selects team or "Personal"
4. Bookmark saved, toast confirmation: "Saved to [Team Name]"
5. Real-time: other team members see new bookmark via WebSocket

### Email Invite Flow

1. Admin/Owner clicks "Invite Member" in team settings
2. InviteModal opens with email input and role selector (Admin/Member)
3. On submit: loading state, then success toast
4. Invitee receives email with "Join Team" CTA button
5. Clicking link: if logged in, joins team; if not, shows "Create account to join"

### Real-time Bookmark Updates

- When team bookmark is added: `team:bookmark:new` WebSocket event
- UI: New bookmark appears at top of team bookmarks list
- Notification: Badge count on team in switcher (if on different page)

| Event | Direction | Data |
|-------|-----------|------|
