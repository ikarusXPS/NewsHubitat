<objective>
Close UAT Test 5 gap — Dashboard NewsCard/SignalCard 3-column grid rendered with vertical row overlap (READ MORE CTA at the bottom of row N covered by the top of row N+1). Root cause is a one-character attribute typo at `apps/web/src/components/virtualization/VirtualizedGrid.tsx:130`: the row wrapper sets `data-row-index={virtualRow.index}` instead of the literal `data-index={virtualRow.index}` that TanStack Virtual v3's `measureElement()` reads to map a measured DOM node back to its row. With the wrong attribute, dynamic remeasurement is silently dropped and every row stays pinned to `estimateSize: 400` — real SignalCard rows often exceed 400px so row N+1 lands ON TOP of row N's tail. Bug predates Phase 40 (originated commit 4551cd45, Phase 35-01, 2026-04-26); first surfaced during Phase 40 UAT because Test 5 was the first time anyone scrolled the Dashboard grid hard enough to notice.

Purpose: Close the visual regression that blocked UAT Test 5 (RelatedPodcasts in NewsCard) by allowing the user to actually see a card without it being clipped by the next row.
Output: One-character rename in VirtualizedGrid.tsx. Optional polish: simplify the inline ref callback to `ref={virtualizer.measureElement}` (the v3 method is bound + stable), and lower estimateSize from 400 → 360 to reduce initial-paint scroll-jump. Both polish items kept inside this same plan since they touch the same file and the same 5-line block — splitting would create artificial busywork.

Debug session evidence: .planning/debug/dashboard-newscard-grid-overlap.md (full root-cause + git blame to 4551cd45 + library doc citation).

Three changes in `apps/web/src/components/virtualization/VirtualizedGrid.tsx`. All targeted at the same component, all mechanically straightforward.

**Change 1 (PRIMARY FIX):** Line 130 — rename `data-row-index` to `data-index`.

Before:
```tsx
data-row-index={virtualRow.index}
```

After:
```tsx
data-index={virtualRow.index}
```

This is the entire fix for the overlap bug. TanStack Virtual v3 reads the literal attribute `data-index` inside `measureElement()` to map the measured DOM node back to its `virtualRow.index`. Any other attribute name causes the lookup to silently fail → no remeasurement → constant 400px estimateSize → overlap.

**Change 2 (POLISH):** Lines 131-133 — simplify the inline ref callback to a direct method reference. `virtualizer.measureElement` is bound + stable in v3.

Before:
```tsx
ref={(el) => {
  if (el) virtualizer.measureElement(el);
}}
```

After:
```tsx
ref={virtualizer.measureElement}
```

This matches `apps/web/src/components/virtualization/VirtualizedList.tsx:102` exactly. Removes a useless inline closure per render.

**Change 3 (POLISH):** Line 44 — lower `estimateSize` from 400 to 360.

Before:
```tsx
estimateSize: useCallback(() => 400, []),
```

After:
```tsx
estimateSize: useCallback(() => 360, []),
```

Rationale (from debug session): SignalCard intrinsic height ≈ 160 (image) + ~150-180 (content+topics+actions) + pb-6 (24px) = ~340-360 base. 360 is closer to the median row size; with measureElement now wired correctly, large cards still get measured precisely. Lower estimate reduces initial scroll-jump on first paint while measurements settle.
