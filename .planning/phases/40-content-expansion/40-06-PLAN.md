Purpose: Premium subscribers get timestamped, searchable transcripts for podcast episodes (lazy on first click) and YouTube/Vimeo videos (free YouTube auto-captions when present, Whisper fallback otherwise) without burning OpenAI credits on duplicate work or on episodes whose publisher already ships a transcript.

- Wired into `RelatedPodcasts` (40-04) and `EmbeddedVideo` (40-05) via "Show transcript" button + `onSeek` callback.
- i18n strings added to all 6 podcasts/videos locale files in DE/EN/FR.
</objective>

     - If `Date.now() - row.lastAttemptedAt.getTime() < 7 * 24 * 3600 * 1000` → return `null` (sentinel still fresh; do not retry per T-40-06-05).
     - Else → fall through to retry the full chain; update `lastAttemptedAt` on completion.
2. **Redis hot-cache check** — `cacheService.get<TranscriptRecord>(\`transcript:${contentType}:${id}\`)`. (Pure read-through; Postgres is the durable layer, Redis is the latency optimisation per D-C1.)
3. For **video contentType only**, attempt `youtubeCaptionService.extract(video.youtubeId)`:
   - If non-null → persist as `provider: 'youtube-captions'`; cache 30 days; return.
4. **Whisper fallback** (both contentTypes):
   - For podcast: load `PodcastEpisode.audioUrl`. For video: skip if non-YouTube (Vimeo lacks captions and we don't transcode Vimeo this phase — return null + sentinel).
   - Call `whisperService.transcribe(audioUrl)`. On success, persist as `provider: 'whisper'`; cache 30 days; return.
5. **Sentinel write on terminal failure** (Whisper threw, or video has no audio source):
   - `prisma.transcript.upsert({ where: { contentType_contentId: { contentType, id } }, create: { ..., provider: 'unavailable', segments: [], fullText: '', language: 'en' }, update: { lastAttemptedAt: new Date() } })`.
   - Return `null`.

`transcribePodcastEpisode(episodeId)` (worker-callable, used by Pitfall 4 path in Task 5):
- Same as `getTranscript('podcast', episodeId)` but bypasses Redis hot-cache (worker writes through to Postgres + Redis).
- If `PodcastEpisode.transcriptUrl` is set: fetch via built-in `fetch`, parse as text. Detect format (`.srt` vs `.vtt` vs JSON) by content-type header AND first 4 chars (`WEBVTT`, digit, `{`); parse to `TranscriptSegment[]`. Persist as `provider: 'publisher-rss'`. NEVER call Whisper.

`searchSegments(contentType, id, query)`:
- Premium-only (route enforces); transcripts are loaded by id then filtered.
- Query Postgres FTS via raw SQL (the Transcript.searchTsv GIN index from 40-01):
  ```typescript
  await prisma.$queryRaw`
    SELECT id, segments, fullText
    FROM "Transcript"
    WHERE "contentType" = ${contentType}
      AND "contentId" = ${id}
      AND search_tsv @@ websearch_to_tsquery('simple', ${query})
  `;
  ```
- Return matched segments only (filter `segments[].text` against tokenized `query` words after FTS narrows to row level — for v1 we only have ONE Transcript per content, so the FTS check tells us "any segment contains?"; segment-level filter happens in JS).

Cache keys:
- `transcript:${contentType}:${id}` → 30 × CACHE_TTL.DAY (D-C1)
- Sentinel: NO Redis entry — Postgres `provider:'unavailable'` is the source of truth (avoids cache stampede if Redis flushes).

Graceful degradation:
- If `cacheService.isAvailable() === false`, skip Redis layers (Postgres still serves). NEVER throw.

import logger from '../utils/logger';

const DAY_IN_MS = 24 * 60 * 60 * 1000;

export class PodcastTranscribeJob {
  private static instance: PodcastTranscribeJob;
  private intervalId: NodeJS.Timeout | null = null;
  private isRunning = false;

  static getInstance(): PodcastTranscribeJob {
    if (!PodcastTranscribeJob.instance) PodcastTranscribeJob.instance = new PodcastTranscribeJob();
    return PodcastTranscribeJob.instance;
  }

  start(): void {
    if (this.isRunning) { logger.warn('podcastTranscribeJob:already_running'); return; }
    this.isRunning = true;
    this.runOnce().catch(err => logger.error('podcastTranscribeJob:initial_failed', err));
    this.intervalId = setInterval(() => {
      this.runOnce().catch(err => logger.error('podcastTranscribeJob:scheduled_failed', err));
    }, DAY_IN_MS);
    logger.info('podcastTranscribeJob:started cadence=daily');
  }

  stop(): void {
    if (this.intervalId) clearInterval(this.intervalId);
    this.intervalId = null;
    this.isRunning = false;
  }

  async runOnce(): Promise<void> {
    logger.info('podcastTranscribeJob:run_start');
    // Find episodes without a Transcript row, ordered by recency, capped to avoid runaway batches
    const episodes = await prisma.podcastEpisode.findMany({
      where: { Transcript: { none: { contentType: 'podcast' } } },  // Prisma relation filter; if Transcript is FK-less, swap to NOT EXISTS subquery via raw
      orderBy: { publishedAt: 'desc' },
      take: 50,  // ceiling per run; ~50 podcasts × $0.36 avg = ~$18 worst-case per nightly batch
    });
    let processed = 0, skippedRss = 0, transcribed = 0, failed = 0;
    for (const ep of episodes) {
      try {
        // PITFALL 4 GUARD: check transcriptUrl FIRST (set by 40-03 RSS poll from <podcast:transcript> tag)
        // Free, often higher quality than auto-generated; skip Whisper entirely.
        // The transcribePodcastEpisode method in TranscriptService handles transcriptUrl internally.
        const result = await TranscriptService.getInstance().transcribePodcastEpisode(ep.id);
        if (result?.provider === 'publisher-rss') skippedRss++;
        else if (result?.provider === 'whisper') transcribed++;
        else failed++;
        if (result) emitTranscriptReady(result.id, 'podcast', ep.id);
      } catch (err) {
        logger.error('podcastTranscribeJob:episode_failed', { episodeId: ep.id, err });
        failed++;
      }
      processed++;
    }
    logger.info('podcastTranscribeJob:run_complete', { processed, skippedRss, transcribed, failed });
  }
}

   export interface TranscriptSegment { startSec: number; endSec: number; text: string }
   export interface TranscriptResponse {
     id: string;
     contentType: 'podcast' | 'video';
     contentId: string;
     language: string;
     segments: TranscriptSegment[];
     provider: 'youtube-captions' | 'whisper' | 'publisher-rss' | 'unavailable';
     transcribedAt: string;

   function getToken(): string {
     if (typeof window === 'undefined') return '';
     return localStorage.getItem('newshub-auth-token') ?? '';

   async function fetchTranscript(contentType: 'podcast' | 'video', id: string): Promise<TranscriptResponse | null> {
     const url = `/api/transcripts/${encodeURIComponent(contentType)}/${encodeURIComponent(id)}`;
     const r = await fetch(url, { headers: { Authorization: `Bearer ${getToken()}` } });
     if (r.status === 404) return null;
     if (!r.ok) throw new Error(`Failed to fetch transcript: ${r.status}`);
     const body = await r.json();
     return body.data as TranscriptResponse;
   }

     id: string,
     options: { enabled?: boolean } = {}
   ) {
     return useQuery({
       queryKey: ['transcript', contentType, id],
       queryFn: () => fetchTranscript(contentType, id),
       staleTime: 60 * 60 * 1000,  // 1h — transcripts are append-mostly; rare invalidation
       enabled: (options.enabled ?? true) && !!id,
       retry: 1,
     });

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/transcripts | Untrusted input crosses here (contentType, id, search query); JWT validated by `authMiddleware`; tier validated by `requireTier('PREMIUM')` |
| Server → OpenAI Whisper API | Outbound; OPENAI_API_KEY in env, never logged or returned |
| Server → YouTube (via `youtube-caption-extractor`) | Outbound; no auth; content treated as untrusted text (rendered by React which auto-escapes) |
| Server → Publisher RSS transcript URL (Pitfall 4 path) | Outbound to arbitrary URLs from `<podcast:transcript>` tag — must validate URL scheme is https; treat content as untrusted |
| YouTube iframe ↔ Browser (postMessage) | Bidirectional message channel; origin must be pinned to `youtube-nocookie.com` / `player.vimeo.com` |
| Worker → Whisper API | Cost-bearing outbound — sentinel rows + 200MB HEAD check are the budget guards |

## STRIDE Threat Register (ASVS L1)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-06-01 | I (Information disclosure) | OPENAI_API_KEY for Whisper | mitigate | Read from `process.env.OPENAI_API_KEY` only inside whisperService; never log; never include in responses; rotate via existing secret-management workflow |
| T-40-06-02 | D (Denial-of-service / cost) | Adversary forces Whisper transcribe of huge files to burn API budget | mitigate | HEAD request before download; reject `content-length > 200MB`; chunk only when needed; per-job cost logged for observability; sentinel row prevents retry storm; manual circuit-breaker possible later (Q-04 default-off) |
| T-40-06-03 | T (Tampering / authz bypass) | Premium-gate bypass via direct API call | mitigate | `requireTier('PREMIUM')` enforced server-side on /api/transcripts/* (mount-level + route-level); frontend gate is UX only; verified by route test asserting 403 with FREE JWT |
| T-40-06-04 | T (Tampering via untrusted iframe message) | YouTube/Vimeo postMessage seek could be spoofed | mitigate | postMessage outbound: pin target origin to `https://www.youtube-nocookie.com` / `https://player.vimeo.com`. Inbound listener: validate `event.origin` against same allowlist before processing |
| T-40-06-05 | D (Sentinel row pollution / retry exhaustion) | Adversary triggers many transcribe failures to fill table or blacklist legitimate retries | mitigate | UNIQUE(contentType, contentId) constraint — sentinel is write-once per content; retry only after `lastAttemptedAt + 7d`; admin can manually clear via `prisma.transcript.delete` |
