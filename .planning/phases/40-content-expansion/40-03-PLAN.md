<objective>
Build the podcast backend (services + config + routes + worker job) on top of the Phase 40-01 schema. After this plan ships, FREE-tier callers can browse curated podcasts, list episodes, fetch a single episode, and request related episodes for any NewsArticle. The worker keeps the curated PodcastEpisode table populated daily and captures the `<podcast:transcript>` Podcasting 2.0 namespace tag (load-bearing for Plan 40-06's transcript orchestrator — silent failure burns Whisper $$).

Purpose: Implements CONT-03 backend per D-B1 (multi-source data: Podcast Index + curated config + iTunes) and D-B2 (entity+topic keyword match, ranked, 24h Redis cache, zero AI calls). Mirrors the Phase 36+37 multi-provider singleton pattern (newsAggregator + aiService + cleanupService) so the next planner reads it as "yet another orchestrator".

**HTML-strip boundary (revision iteration 1 — H8 + M2 fix):** RSS feed items frequently embed HTML (`<p>`, `<strong>`, `<a>`, etc.) inside `description`/`content`/`contentSnippet`. Storing raw HTML and letting React render it auto-escaped causes every `<PodcastEpisodeCard>` (planned in 40-04) to display literal markup like `&lt;p&gt;Hello&lt;/p&gt;` — broken UX on every card. The cleanest fix is at the persistence boundary in this plan's `pollFeed()`: strip once with cheerio (already an `apps/web` dep, already used by `newsCrawler.ts` and `stealthScraper.ts`), store clean text, and every downstream consumer (PodcastEpisodeCard, RelatedPodcasts, future transcript-excerpt search, RAG context window) gets plain text without needing DOMPurify or a sanitizer on the frontend (DOMPurify is not in the standard stack per 40-04 T-40-04-02).

- 4 new services (podcastIndex, itunesPodcast, podcastMatcher, podcastService) + their unit tests
- 1 new config (podcasts.ts, >=30 curated feeds)
- 1 new Express router with 4 routes + Zod schemas
- 1 new worker job (podcastFeedPollJob) + unit test

<podcast_index_hmac_spec>
<!-- From RESEARCH.md "Podcast Index HMAC auth (D-B1)" + verified at https://podcastindex-org.github.io/docs-api/ -->
- Base URL: `https://api.podcastindex.org/api/1.0`
- Required headers per request:
  - `User-Agent: NewsHub/2.0 (https://newshub.example)`
  - `X-Auth-Date: <unix-time-seconds>` (string)
  - `X-Auth-Key: <PODCAST_INDEX_API_KEY>`
  - `Authorization: <sha1(KEY + SECRET + unixTime), lowercase hex>`
- Time window: server accepts +/- 3 minutes
- Search-by-term endpoint: `GET /search/byterm?q=<query>&max=<n>&fulltext`
- Search-episode endpoint: `GET /search/byperson` and `GET /search/byterm` (episodes via `/episodes/byid` + `/episodes/byfeedid`)
- Use `crypto.createHash('sha1').update(KEY + SECRET + unixTime).digest('hex')` (Node built-in `crypto`)
</podcast_index_hmac_spec>

<rank_formula>
<!-- From RESEARCH.md B5 + CONTEXT.md D-B2 -->
For each candidate episode `e` matched against article `a`:
  score(e, a) =
      10 * (# of a.entities that appear in e.title OR e.description) +
       3 * (# of a.topics that appear in e.title OR e.description) +
       2 * recencyScore(e.publishedAt)         // exp decay: 1.0 if <7d old, 0.5 if 30d, 0.1 if 365d
     + 1 * popularityScore(e)                  // log10(1 + e.podcast.subscribers||1) when source provides it; else 0

Take top 5 episodes after dedupe on `podcastGuid` then `hash(podcastTitle + episodeTitle + publishedAt)`.
String matching is case-insensitive substring match against `(e.title + ' ' + e.description).toLowerCase()`. No stemming this phase (deferred per CONTEXT.md "Deferred Ideas" — embeddings).

NOTE: The matcher consumes the *plain-text* description that was already stripped at write-time. Substring match against raw HTML would yield false negatives when entity words are wrapped in tags (e.g. `<a href="...">Trump</a>` would not match `"trump"` substring naively if the search included tag chars). Stripping at the persistence boundary fixes this for the matcher AND every other consumer in one place.
</rank_formula>

<html_strip_spec>
<!-- Revision iteration 1 — H8 (HIGH) + M2 (MEDIUM) fix from plan-checker -->
RSS feed items frequently embed HTML in `description` / `content` / `contentSnippet`. The plain-text fix lives in `podcastService.ts` (the file that owns `pollFeed`):

```typescript
import * as cheerio from 'cheerio';

/**
 * Strip HTML at the persistence boundary so all downstream consumers
 * (PodcastEpisodeCard, RelatedPodcasts, transcript-excerpt search, RAG)
 * work with clean text. Avoids requiring DOMPurify on the frontend
 * (not in standard stack per 40-04 T-40-04-02). cheerio handles malformed
 * HTML gracefully — same library + import shape used by newsCrawler.ts
 * and stealthScraper.ts.
 */
function stripHtml(html: string | undefined | null): string {
  if (!html) return '';
  const $ = cheerio.load(html, { decodeEntities: true });
  return $.root().text().trim().replace(/\s+/g, ' ');
}
```

Apply to `description` (load-bearing — RSS sources almost always wrap descriptions in `<p>`/`<a>`) and to `title` (defensive — titles are usually plain but a small minority of feeds wrap them in `<![CDATA[<b>...</b>]]>`).

Round-trip examples (asserted in Task 4 test):
- Input `'<p>Hello <strong>world</strong></p>'` -> `'Hello world'`
- Input `'   foo &amp; bar   '` -> `'foo & bar'`
- Input `'<p>line 1</p><p>line 2</p>'` -> `'line 1 line 2'`  (collapsed via `\s+` regex)
- Input `undefined` -> `''`
- Input `''` -> `''`
- Input malformed `'<p>unclosed <strong>nested'` -> `'unclosed nested'`  (cheerio tolerates malformed HTML)
</html_strip_spec>

       export function rankEpisodes(
         article: { entities: string[]; topics: string[] },
         candidates: CandidateEpisode[],
       ): MatchedEpisode[] {
         const ents = (article.entities ?? []).slice(0, 3);  // CONTEXT.md <specifics> top-3 entities
         const tops = (article.topics ?? []).slice(0, 2);    // top-2 topics
         const scored = candidates.map(c => {
           const hay = `${c.episodeTitle} ${c.description}`;
           const eHits = ents.filter(e => containsCI(hay, e)).length;
           const tHits = tops.filter(t => containsCI(hay, t)).length;
           const score = 10 * eHits + 3 * tHits + 2 * recencyScore(c.publishedAt) + popularityScore(c.subscribers);
           return { ...c, score };
         }).filter(m => m.score > 0);

         // Dedupe — keep highest-scoring per dedupe key
         const byKey = new Map<string, MatchedEpisode>();
         for (const m of scored) {
           const k = dedupeKey(m);
           const existing = byKey.get(k);
           if (!existing || m.score > existing.score) byKey.set(k, m);
         }

         return [...byKey.values()].sort((a, b) => b.score - a.score).slice(0, 5);

     * Podcast Feed Poll Job (Phase 40-03 / CONT-03)
     * Daily worker that polls each curated podcast feed for new episodes.
     * Captures the Podcasting 2.0 <podcast:transcript> tag (Pitfall 4 — load-bearing
     * for the Phase 40-06 transcript orchestrator's cost model).
     *
     * The per-feed work — including HTML stripping at the persistence boundary
     * (H8/M2 fix) — happens inside podcastService.pollFeed(). This job is just
     * the iterator + retry shell.
     *
     * Gated by RUN_JOBS=true (Phase 37 contract) — only the app-worker Swarm
     * replica starts this job; web replicas do not.
     */

    import { PodcastService } from '../services/podcastService';
    import { PODCAST_FEEDS } from '../config/podcasts';
    import logger from '../utils/logger';

    const DAY_IN_MS = 24 * 60 * 60 * 1000;
    const POLL_INTERVAL_MS = DAY_IN_MS;  // Q-01: nightly batch

    export class PodcastFeedPollJob {
      private static instance: PodcastFeedPollJob;
      private intervalId: NodeJS.Timeout | null = null;
      private isRunning = false;
      private podcastService = PodcastService.getInstance();

      private constructor() {
        logger.info('PodcastFeedPollJob initialized');
      }
      static getInstance(): PodcastFeedPollJob {
        if (!PodcastFeedPollJob.instance) PodcastFeedPollJob.instance = new PodcastFeedPollJob();
        return PodcastFeedPollJob.instance;
      }

      start(): void {
        if (this.isRunning) { logger.warn('PodcastFeedPollJob already running'); return; }
        this.isRunning = true;
        this.runOnce().catch(err => logger.error('Initial podcast feed poll failed:', err));
        this.intervalId = setInterval(() => {
          this.runOnce().catch(err => logger.error('Scheduled podcast feed poll failed:', err));
        }, POLL_INTERVAL_MS);
        logger.info('PodcastFeedPollJob started — runs daily');
      }

      stop(): void {
        if (this.intervalId) { clearInterval(this.intervalId); this.intervalId = null; }
        this.isRunning = false;
        logger.info('PodcastFeedPollJob stopped');
      }

      async runOnce(): Promise<{ totalInserted: number }> {
        logger.info(`podcast-feed-poll:start (${PODCAST_FEEDS.length} feeds)`);
        let totalInserted = 0;
        for (const feed of PODCAST_FEEDS) {
          try {
            const inserted = await this.podcastService.pollFeed(feed);
            totalInserted += inserted;
          } catch (err) {
            logger.warn(`pollFeed(${feed.id}) error: ${(err as Error).message}`);
          }
        }
        logger.info(`podcast-feed-poll:done (inserted=${totalInserted})`);
        return { totalInserted };
      }
    }
