<objective>
Build the Phase 40 podcast frontend so users can discover, browse, and play news-relevant podcast episodes from two surfaces:

1. **Inline in `NewsCard`** — a collapsed-by-default "Related podcast episodes" section that lazy-fetches matches for the current article and shows 1-3 episodes with a vanilla-audio player (per CONTEXT D-B3 + RESEARCH B4).
2. **Dedicated `/podcasts` page** — browse the curated `apps/web/server/config/podcasts.ts` feed list (shipped by 40-03), drill into episode lists, and search across podcasts/episodes. Premium users additionally get a transcript-excerpt search toggle (Q-05); FREE users on web see `<UpgradePrompt inline>`, FREE users on mobile see plain text per CC-01 + Phase 39 D-09.

This plan is the consumer of the 40-03 backend (`/api/podcasts/*` routes). It does NOT introduce a transcript drawer — that lives in 40-06 and mounts inside the components this plan creates.

Purpose: Close requirement **CONT-03** ("user can browse podcast episodes related to news topics") with a UX that respects the dark-cyber design system, the 250 KB bundle warning, the LCP < 2s budget, and the reader-app exemption.

Output: 7 new components/hooks, 6 modified files (NewsCard + Sidebar + App + routes + i18n init + 3 locale JSONs filled), 5 component test files, and a green `pnpm typecheck && pnpm test:run && pnpm build`.
</objective>

### Backend endpoints owned by 40-03 (already live by the time this plan runs)

```
GET /api/podcasts                               -> { success, data: PodcastFeed[] }
GET /api/podcasts/:feedId/episodes              -> { success, data: PodcastEpisode[], meta: { total, page, limit, hasMore } }
GET /api/podcasts/episodes/:id                  -> { success, data: PodcastEpisode }
GET /api/podcasts/related/:articleId            -> { success, data: MatchedEpisode[] }       // 24h Redis cache
GET /api/podcasts/search?q=...&country=...      -> { success, data: PodcastEpisode[] }       // FREE tier
GET /api/podcasts/transcripts/search?q=...      -> { success, data: TranscriptHit[] }        // 40-06 wires this; requireTier('PREMIUM')
```

All FREE-tier endpoints respond with `Cache-Control: public, max-age=300` and the standard envelope `{ success, data, error?, meta? }`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → backend (`/api/podcasts/*`) | Article ID + search query strings cross here; both validated server-side by 40-03 |
| RSS publisher → audio element | `episode.audioUrl` originates from podcast RSS feeds parsed by 40-03; audio bytes cross from external CDNs into the user's browser |
| RSS publisher → DOM | `episode.description` may contain HTML from RSS; this plan handles it as plain text only |
| Premium gate (web) | `subscriptionTier` read from `useAuth()` (JWT-backed); UI gates transcript-search; server-side gate is the load-bearing security control (40-06) |
| Premium gate (mobile) | Same JWT origin; UI shows plain text per Apple Rule 3.1.1(a) |
