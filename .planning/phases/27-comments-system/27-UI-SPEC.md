## Interaction Patterns

### New Comment Flow
1. User types in CommentInput
2. Typing indicator broadcasts to other viewers (300ms debounce)
3. User clicks "Post Comment"
4. AI toxicity check runs (async, <500ms typical)
5. If passed: Comment auto-publishes, slides in with animation
6. If flagged: Toast shows "flagged for review", comment still visible with badge
7. Success toast: "Comment posted"

### Real-time Updates (per D-12)
- New comments: Auto-insert with slide animation (Framer Motion)
- Edits/deletes: Do NOT auto-update (refresh required)
- Typing indicator: Show "Someone is typing..." without usernames

### Reply Flow
1. User clicks "Reply" on a comment
2. Inline CommentInput appears below the comment
3. Reply posts as child with 24px indent
4. Deeper replies link to parent (no nesting beyond 2 levels per D-05)

### Edit Flow (per D-08)
1. Edit button visible for 15 minutes after posting
2. Click reveals inline edit textarea
3. Save updates comment, adds "(edited)" indicator
4. Cancel reverts to original

### Delete Flow (per D-09)
1. User clicks "Delete" on their comment
2. ConfirmDialog appears with warning
3. On confirm: Soft delete, text replaced with "[Comment deleted]"
4. Replies remain visible under deleted parent

### Flag Flow
1. User clicks "Flag" on any comment
2. Comment gets `isFlagged: true` with reason
3. Visual badge appears: "Flagged for review"
4. Comment remains visible (per D-03)
5. Toast confirms: "Comment flagged for review"

## Animation Specifications

| Animation | Library | Specification |
|-----------|---------|---------------|
| New comment slide-in | Framer Motion | `initial={{ opacity: 0, y: -20 }}`, `animate={{ opacity: 1, y: 0 }}`, duration 0.3s |
| Reply expand/collapse | Framer Motion | `AnimatePresence` with height animation |
| Typing indicator dots | CSS | 3-dot pulse keyframes, 1s loop |
| Delete fade-out | Framer Motion | `exit={{ opacity: 0, scale: 0.95 }}`, duration 0.2s |
| Input focus glow | CSS | `box-shadow: 0 0 20px rgba(0, 240, 255, 0.2)` |

| Breakpoint | Behavior |
|------------|----------|
| Mobile (<768px) | Full-width cards, stacked layout, bottom-sheet for delete confirm |
| Tablet (768px-1024px) | Same as mobile |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Focus management | Auto-focus textarea on reply click |
| Keyboard navigation | Tab through comments, Enter to expand replies |
| Screen reader | aria-label on action buttons, live region for new comments |
| Color contrast | All text meets WCAG AA (4.5:1 minimum) |
| Motion preference | Respect `prefers-reduced-motion` for animations |
