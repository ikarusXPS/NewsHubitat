### Task 2: GREEN — Handler fix + structured logging (commit `2896766`)

**EDIT 1 — Structured error log (`apps/web/server/routes/webhooks/stripe.ts:72`):**

```typescript
// BEFORE (broken — empty log when err lacks .message):
logger.error('[Webhook] Processing error:', err);

// AFTER (T-36.5-01 mitigated):
logger.error(`[Webhook] Processing error: ${JSON.stringify({
  message: err instanceof Error ? err.message || '<empty>' : String(err),
  name: err instanceof Error ? err.name : typeof err,
  stack: err instanceof Error ? err.stack : undefined,
  code: (err as { code?: string })?.code,
  eventId: event.id,
  eventType: event.type,
})}`);
```

**EDIT 2 — Defensive price item guard (`apps/web/server/services/stripeWebhookService.ts:190-199`):**

Root cause: Shape B — `subscription.items.data[0]?.price.id` optional-chains `data[0]` but NOT `price`. If `data[0]` exists with `price: undefined`, this throws `TypeError: Cannot read properties of undefined (reading 'id')`. The TypeError's `.message` is empty on some Node builds, producing the empty error log observed in production.

```typescript
// Guard added at top of handleSubscriptionUpdated:
const priceItem = subscription.items?.data?.[0];
if (!priceItem?.price?.id) {
  logger.warn(`[Webhook] subscription.updated: Missing price on subscription ${subscription.id} — skipping tier update`);
  return;
}
```

This returns cleanly (no throw) so the idempotency layer marks the event as processed and Stripe does not retry.
