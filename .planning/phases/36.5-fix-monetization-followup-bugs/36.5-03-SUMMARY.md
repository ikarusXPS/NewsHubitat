## Decisions Made

- **Non-optional `subscriptionTier` field.** The plan's CONTEXT.md D-03 specified non-optional and the Prisma schema has `@default(FREE)` so the wire payload always carries it. Optional `?` would force every consumer to pessimistically check undefined.
- **Kept existing redundant casts.** `Pricing.tsx:151` and `AIUsageCounter.tsx:23` still use `(user as { subscriptionTier?: string } | null)?.subscriptionTier`. After widening, the cast is a downcast TO a wider type — TypeScript still allows it and `pnpm typecheck` exits 0. The plan called this OPTIONAL cleanup; deferring rather than expanding scope.
- **No external conditional guard.** Did not wrap the Sidebar `<SubscriptionBadge>` call in `{user?.subscriptionTier !== 'FREE' && ...}` — `SubscriptionBadge.tsx:42` already returns null for FREE. Adding an outer guard would duplicate the gate and risk drift.
