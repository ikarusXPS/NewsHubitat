**Workflow triggers** (per D-02):
```yaml
on:
  pull_request:
  push:
    branches: [main]
```

**Job 1: lint** (runs in parallel)
- Runner: `ubuntu-latest` (D-04)
- Steps: checkout → setup-node (Node 25, cache: npm per D-05) → npm ci → npm run lint
- Cache key: `~/.npm` for dependency caching (D-05)

**Job 2: typecheck** (runs in parallel)
- Same setup as lint job
- Run: `npm run typecheck`

**Job 3: test** (runs in parallel, with service containers per D-06)
- Service containers (D-06):
  - `postgres`: image `postgres:17`, env vars POSTGRES_USER=newshub, POSTGRES_PASSWORD=test, POSTGRES_DB=newshub, health check `pg_isready`, ports `5432:5432`
  - `redis`: image `redis:7.4-alpine`, health check `redis-cli ping`, ports `6379:6379`
- Steps: checkout → setup-node → npm ci → npx prisma generate → npx prisma db push → npm run test:coverage
- Environment variables: DATABASE_URL=postgresql://newshub:test@localhost:5432/newshub, REDIS_URL=redis://localhost:6379
- Coverage threshold enforcement: 80% (D-13, matches vitest.config.ts)

**Job 4: build** (depends on [lint, typecheck, test])
- Steps: checkout → setup-buildx → login to GHCR (D-07) → build-push-action
- GHCR authentication: registry `ghcr.io`, username `${{ github.actor }}`, password `${{ secrets.GITHUB_TOKEN }}`
- Build action (D-08 tags):
  - context: `.`
  - push: `${{ github.ref == 'refs/heads/main' }}` (only push on main branch)
  - tags: `ghcr.io/${{ github.repository }}:${{ github.sha }}`, `ghcr.io/${{ github.repository }}:latest`, `ghcr.io/${{ github.repository }}:${{ github.ref_name }}`
  - cache-from: `type=gha`
  - cache-to: `type=gha,mode=max`
- Permissions: `contents: read`, `packages: write`

**Job 5: e2e** (depends on build)
- Service containers: same postgres/redis as test job
- Steps: checkout → setup-node → npm ci → npx prisma generate → npx prisma db push → npx playwright install --with-deps chromium (D-14) → npx playwright test
- Environment: CI=true (auto-set by GitHub Actions)
- Upload artifact: playwright-report on failure (`if: always()`)

**Linear job chain** (per D-03): lint/typecheck/test run in parallel → build → e2e

Pin all GitHub Actions to commit SHAs (not version tags) per RESEARCH.md anti-pattern to prevent March 2026-style supply chain attacks:

Use `setup-node` with `cache: 'npm'` for automatic `~/.npm` caching (D-05).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → GHCR.io | Runner authenticates to registry with GITHUB_TOKEN |
| Workflow → External actions | Third-party actions (docker/*, actions/*) run with repo access |
| PR author → CI environment | PR code runs in CI with access to secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering | GitHub Actions (version tags) | mitigate | Pin all actions to commit SHAs (not version tags) to prevent March 2026-style supply chain attacks. Use actions/checkout@b4ffde65, not @v5. |
| T-18-02 | Information Disclosure | GITHUB_TOKEN in logs | mitigate | Use `${{ secrets.GITHUB_TOKEN }}` (automatic masking in logs). Never echo or print token values. |
| T-18-03 | Elevation of Privilege | Docker build accessing secrets | mitigate | Use `permissions: packages: write` explicit grant. GITHUB_TOKEN auto-scoped to repository. No cross-repo access. |
| T-18-04 | Denial of Service | Unbounded workflow runs | accept | GitHub Actions has built-in concurrency limits (2000 min/month free tier). No additional mitigation needed. |
| T-18-05 | Tampering | Malicious PR code in CI | mitigate | PR builds don't push images (`push: ${{ github.ref == 'refs/heads/main' }}`). Quality gates fail on malicious code. |
</threat_model>
