## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] DE pricing.json restored to UTF-8 umlauts**
- **Found during:** Task 3 PART B (locale recovery)
- **Issue:** The bde8ee4 source DE pricing.json uses ASCII transliterations (`Waehle deinen Plan`, `Jaehrlich`, `geniesse`, `Faehigkeiten`, `Loesungen`) instead of proper umlauts. This was likely a result of a Windows console codepage issue at the time the locale was authored. The plan's success criterion explicitly requires the umlaut-correct string `Wähle deinen Plan`, and the existing apps/web DE locale precedent (e.g., common.json `Löschen`, `Überprüfung`) uses proper umlauts.
- **Fix:** Wrote the DE pricing.json with proper UTF-8 umlauts: `Wähle`, `Jährlich`, `Fähigkeiten`, `Lösungen`, `Genieße`, `überprüft`, `Prioritäts`, `Geld-zurück`. All semantic content byte-stable with bde8ee4; only character encoding corrected.
- **Files modified:** apps/web/public/locales/de/pricing.json
- **Commit:** 6a0580e

**2. [Rule 1 - Bug] FR pricing.json restored to proper French accents**
- **Found during:** Task 3 PART B
- **Issue:** Same root cause as #1 — bde8ee4 FR pricing.json was authored without French accents (`Debloquez`, `evenements`, `securise`, `Acces illimite`, `Verification`, `personnalisees`).
- **Fix:** Wrote FR pricing.json with proper accents: `Débloquez`, `événements`, `sécurisé`, `Accès illimité`, `Vérification`, `personnalisées`, `Économisez`, `équipes`, `régions`, `mises à jour`, `préréglages`, etc. Semantic content unchanged.
- **Files modified:** apps/web/public/locales/fr/pricing.json
- **Commit:** 6a0580e

**3. [Rule 1 - Bug] DE common.json upgrade/ai keys restored to UTF-8 umlauts**
- **Found during:** Task 3 PART B
- **Issue:** bde8ee4 DE common.json appended `upgrade.unlockFeature: "Upgrade fuer {{feature}}"` and `ai.dailyQueries: "Taegliche KI-Anfragen"` / `ai.upgradeCta: "Upgrade fuer unbegrenzte Anfragen"` — same ASCII transliteration issue.
- **Fix:** Appended the keys with proper umlauts: `Tägliche`, `für`. Translated `unlockFeature` to natural German `"Upgrade um {{feature}} freizuschalten"` (the bde8ee4 form `Upgrade fuer {{feature}}` reads ungrammatical in German; this is a minor wording correction beyond pure ASCII fix).
- **Files modified:** apps/web/public/locales/de/common.json
- **Commit:** 6a0580e

### Rule 3 Fix

**4. [Rule 3 - Blocking issue] Zustand language union widening**
- **Found during:** Task 3 PART A
- **Issue:** After updating i18n.ts line 40 to accept `lng === 'fr'`, the `validLng` is typed as `'de'|'en'|'fr'`. But Zustand's `setLanguage` signature was `(lang: 'de' | 'en') => void`, causing a TS narrowing error.
- **Fix:** Widened both `AppState.language` and `setLanguage` parameter type in apps/web/src/store/index.ts from `'de' | 'en'` to `'de' | 'en' | 'fr'` (single 2-line change). Plan anticipated this in the deviation_handling block.
- **Files modified:** apps/web/src/store/index.ts
- **Commit:** 6a0580e
