## Deviations from Plan

### Adapted mock-access pattern

The plan's scaffold (lines 161-162) used:
```typescript
const { prisma } = await import('../db/prisma');
const { CacheService } = await import('../services/cacheService');
```

This relies on top-level `await` and breaks `vi.mocked(...)` typing in some vitest configs. I substituted the project's existing analog pattern from `subscriptionService.test.ts` (lines 90-92):
```typescript
import { prisma } from '../db/prisma';
import { CacheService } from '../services/cacheService';
// ...later, inside tests:
vi.mocked(prisma.user.findUnique).mockResolvedValue(...);
vi.mocked(CacheService.getInstance).mockReturnValue(...);
```

Result: clean typing, no test-runner gymnastics, fully aligned with existing patterns. No functional change to coverage or assertions.

### Tests added beyond plan minimums

Plan required 9 tests; SUMMARY ships 15. The 6 extras strengthen contract coverage:
1. `attachUserTier with req.user (cache miss) -> tier from prisma` — proves attachUserTier reads the same cache key as requireTier
2. `requireTier - DB has no user record -> 401` — covers the prisma-returns-null branch (line 60-65 of requireTier.ts)
3. `requireTier - warm cache short-circuits prisma` — proves the 5-min cache TTL behavior
4. `hasTier(FREE, FREE) returns true` — covers equal-tier comparison
5. `attachUserTier never sends a response` — explicit non-enforcement assertion
6. `mockNext.toHaveBeenCalledTimes(1)` (vs plan's `toHaveBeenCalled`) — guards against accidental double-invocation regressions

These are pure additions; they do not modify the implementation contract.
