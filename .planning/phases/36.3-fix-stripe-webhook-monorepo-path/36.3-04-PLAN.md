   b. Capture pre-retry row count for `checkout.session.completed`:
      ```bash
      PRE_COUNT=$(psql postgresql://newshub:newshub_dev@localhost:5433/newshub -tAc \
        'SELECT COUNT(*) FROM "ProcessedWebhookEvent" WHERE "eventType" = '"'"'checkout.session.completed'"'"';')
      echo "pre-retry count: $PRE_COUNT"
      ```

   c. Re-trigger the same event:
      ```bash
      stripe trigger checkout.session.completed
      sleep 5
      ```

      Each `stripe trigger` issues a NEW Stripe event ID, so the second `checkout.session.completed` will create a NEW row legitimately (not a duplicate). The idempotency check is for THE SAME event ID being processed twice — which happens when Stripe retries due to network failure or our backend timing out.

      To test true idempotency, we need to replay an event with the same ID. The Stripe CLI offers this via `stripe events resend <event-id>`:
      ```bash
      # Capture an event ID from a recent trigger (use the latest checkout.session.completed)
      EVT_ID=$(stripe events list --limit 1 --type checkout.session.completed -q | grep -oE 'evt_[A-Za-z0-9]+' | head -1)
      echo "Replaying event: $EVT_ID"

      # Get current row count for this specific event id
      PRE_REPLAY=$(psql postgresql://newshub:newshub_dev@localhost:5433/newshub -tAc \
        "SELECT COUNT(*) FROM \"ProcessedWebhookEvent\" WHERE \"id\" = '$EVT_ID';")
      echo "pre-replay row count for $EVT_ID: $PRE_REPLAY"

      stripe events resend "$EVT_ID"
      sleep 5

      POST_REPLAY=$(psql postgresql://newshub:newshub_dev@localhost:5433/newshub -tAc \
        "SELECT COUNT(*) FROM \"ProcessedWebhookEvent\" WHERE \"id\" = '$EVT_ID';")
      echo "post-replay row count for $EVT_ID: $POST_REPLAY"

      if [ "$PRE_REPLAY" = "$POST_REPLAY" ]; then
        echo "IDEMPOTENCY PASS: row count unchanged ($PRE_REPLAY -> $POST_REPLAY)"
      else
        echo "IDEMPOTENCY FAIL: row count changed ($PRE_REPLAY -> $POST_REPLAY)"
        exit 1
      fi
      ```

      Append the idempotency proof to the DB query log:
      ```bash
      cat >> .planning/phases/36.3-fix-stripe-webhook-monorepo-path/36.3-04-db-query.log <<EOF

=== Idempotency replay test ===
Event ID replayed: $EVT_ID
Pre-replay rows for that event ID: $PRE_REPLAY
Post-replay rows for that event ID: $POST_REPLAY
IDEMPOTENCY: $([ "$PRE_REPLAY" = "$POST_REPLAY" ] && echo "PASS" || echo "FAIL")
EOF
      ```

   d. Tear down the background processes:

      ```bash
      kill $STRIPE_PID 2>/dev/null || true
      kill $BACKEND_PID 2>/dev/null || true
      wait $STRIPE_PID 2>/dev/null || true
      wait $BACKEND_PID 2>/dev/null || true
      ```
