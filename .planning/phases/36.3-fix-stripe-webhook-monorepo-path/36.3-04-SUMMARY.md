## Decisions Made

1. **Accept 7-of-8 direct event coverage; the 8th covered structurally.** `customer.subscription.resumed` cannot be triggered via `stripe trigger` (CLI does not support that event) and reaching `status=paused -> resumed` via API requires a real payment-failure cycle (infeasible in a synthetic-fixture sandbox session). Plan 04's purpose is proving the relocated route is hit end-to-end; the handler dispatches all 8 events through the same switch and the same `processWebhookIdempotently` path, exercised 93 times across 31 event types. Reaching the 8th event in production is a `production-readiness` phase responsibility once a real webhook URL is registered.
2. **Force-add the two `.log` files.** `.gitignore:3:*.log` is a project-wide rule for runtime/dev logs and should not be amended for one-shot verification evidence; `git add -f` is the correct narrow override.
3. **Two-commit split.** Verification artifacts (`test(36.3-04)`) and plan metadata (`docs(36.3-04)`) are semantically different concerns. Keeping them separate makes either independently revertable.
