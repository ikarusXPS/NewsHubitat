## Interaction Contracts

### Share Button Click (Desktop)

1. User clicks platform icon
2. Track click via fire-and-forget mutation (`POST /api/share/:code/click`)
3. Open platform URL in new tab immediately (`window.open(url, '_blank', 'noopener,noreferrer')`)
4. Button shows brief scale animation (framer-motion: `scale: 0.95` for 100ms)

**Critical:** Never await tracking before opening URL (per RESEARCH.md anti-pattern).

### Copy Link Click

1. User clicks Copy icon
2. Copy share URL to clipboard via `navigator.clipboard.writeText()`
3. Icon transforms: Copy -> Check (green) for 2000ms
4. Toast notification: "Link copied!" (success variant)
5. Track click with platform = 'copy'

### Native Share (Mobile)

1. User taps Share button
2. Call `navigator.share({ title, url })`
3. If successful: track with platform = 'native'
4. If user cancels: AbortError, no action needed
5. If API fails: show platform icon fallback row

### My Shares Load

1. Component mounts in Profile page
2. Fetch user shares via `GET /api/share/my` (auth required)
3. Loading: skeleton rows (3 rows)
4. Success: render table with analytics
5. Empty: show empty state message
6. Error: graceful degradation (return null, don't break page)

## Animation Specifications

| Element | Trigger | Animation |
|---------|---------|-----------|
| Platform button | Click | `scale: [1, 0.95, 1]`, duration: 150ms, ease: spring |
| Copy icon -> Check | Success | `opacity: [0, 1]`, `rotate: [0, 360]`, duration: 300ms |
| Check -> Copy | After 2s | `opacity: [1, 0, 1]` (crossfade), duration: 200ms |
| My Shares row | Appear | `opacity: [0, 1]`, `y: [10, 0]`, stagger: 50ms |
| Empty state | Appear | `opacity: [0, 1]`, `scale: [0.95, 1]`, duration: 300ms |

## Error States

| Error | User Message | Visual |
|-------|--------------|--------|
| Share tracking failed | (silent, fire-and-forget) | No visual, log to console |
