**Pattern to copy:**

**Target structure (Claude drafts; user reviews per CONTEXT line 61):**
```markdown
# Production Migration Note — Phase 36.2 Enum Conversion

**Status:** Not executed in 36.2. Dev DB only.
**Owner of execution:** Production-readiness phase (TBD).

## What 36.2 Did (Dev Only)

`prisma db push --accept-data-loss` converted `User.subscriptionTier` and
`User.subscriptionStatus` from `String?` to Prisma enums (`SubscriptionTier`,
`SubscriptionStatus`). On the dev DB the existing 5 columns held `NULL` for
all users (no real Stripe data), so the conversion was lossless.

## What Production Requires

Production may have non-null `text` values in these columns. `db push` is
unsafe — Prisma 7 will drop and recreate the columns.

A hand-written SQL migration is required:

\`\`\`sql
ALTER TABLE "User"
  ALTER COLUMN "subscriptionTier" TYPE "SubscriptionTier"
    USING "subscriptionTier"::text::"SubscriptionTier",
  ALTER COLUMN "subscriptionStatus" TYPE "SubscriptionStatus"
    USING "subscriptionStatus"::text::"SubscriptionStatus";
\`\`\`

**Pre-requisites:**
```
