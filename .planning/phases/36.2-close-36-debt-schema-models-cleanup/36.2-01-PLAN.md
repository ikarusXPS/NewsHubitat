## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36.2-01 | Tampering | ProcessedWebhookEvent.id (caller-supplied String @id) | accept | The Stripe event ID is supplied by the Stripe webhook delivery, not user input. Forging requires bypassing HMAC signature verification (handled in 36.3); the @id constraint at the DB layer enforces uniqueness as a defense-in-depth backup against duplicate processing. Severity: LOW. |
| T-36.2-02 | Information Disclosure | User.referralCode is @unique and exposed to other users at signup | accept | Referral code is a low-value identifier (not a credential, no auth bypass). Per security_threat_model_required note in planning context. Severity: LOW. |
| T-36.2-03 | Denial of Service | ReferralReward FK Cascade allows mass-delete via account deletion | accept | Cascade is intentional per D-04 — referral history follows the user. Account deletion is rate-limited at the auth layer (existing AUTH controls). Severity: LOW. |
| T-36.2-04 | Tampering | StudentVerification.status changeable without auth | mitigate (deferred) | No service code in 36.2 writes this column. The future student-verification flow (deferred phase) must enforce that only admins can set status to "approved". 36.2 only declares the schema. Severity: LOW (no consumer yet). |
| T-36.2-05 | Elevation of Privilege | subscriptionTier @default(FREE) on row insert | accept | New users default to FREE — same as the prior `String?` + `?? 'FREE'` semantics. No EoP risk. Severity: LOW. |

No HIGH-severity threats. All threats accepted or deferred to consumer-code phases. ASVS L1 baseline satisfied.
