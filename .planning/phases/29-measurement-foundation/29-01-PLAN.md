  <action>
1. Add `pull-requests: write` to permissions block (required for compressed-size-action to post comments):
   ```yaml
   permissions:
     contents: read
     packages: write
     pull-requests: write
   ```

2. Add new job `bundle-analysis` AFTER the `test` job and BEFORE the `build` job:
   ```yaml
   bundle-analysis:
     name: Bundle Analysis
     runs-on: ubuntu-latest
     needs: [lint, typecheck, test]
     steps:
       - name: Checkout
         uses: actions/checkout@v4

       - name: Setup Node.js
         uses: actions/setup-node@v4
         with:
           node-version: 22
           cache: 'npm'

       - name: Install dependencies
         run: npm ci --legacy-peer-deps

       - name: Generate Prisma client
         run: npx prisma generate

       - name: Build with visualizer
         run: npm run build

       - name: Upload bundle stats
         uses: actions/upload-artifact@v4
         with:
           name: bundle-stats
           path: dist/stats.html
           retention-days: 30

       - name: Report bundle size
         uses: preactjs/compressed-size-action@v2
         with:
           repo-token: ${{ secrets.GITHUB_TOKEN }}
           pattern: './dist/**/*.{js,css}'
           compression: 'brotli'
   ```

Note per D-04: compressed-size-action posts PR comments but does NOT block PRs on size increase. Team reviews and decides on acceptable growth.
  </action>
