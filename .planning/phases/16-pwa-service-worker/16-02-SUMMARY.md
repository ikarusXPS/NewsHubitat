# Phase 16 Plan 02: Service Worker Registration Summary

**One-liner:** Production service worker registration with workbox-window lifecycle handling and update prompts

## What Was Built

Implemented service worker registration infrastructure for production builds, enabling PWA update detection and skip-waiting functionality. Users now receive update prompts when new versions are available.

### Deliverables

1. **useServiceWorker Hook** (src/hooks/useServiceWorker.ts)
   - Service worker registration wrapper using Workbox Window API
   - State management for registration status, update detection, and errors
   - Event listeners for `waiting` and `activated` lifecycle events
   - `updateServiceWorker()` function to trigger skip waiting and reload
   - Production-only registration (skips in dev mode)

2. **Production Bootstrap** (src/main.tsx)
   - Dynamic import of workbox-window in production builds only
   - Service worker registration before React createRoot
   - Update prompt with browser confirm dialog
   - Skip waiting and reload on user confirmation

### Architecture

```
main.tsx (production check)
  ↓ (dynamic import)
workbox-window
  ↓ (registers)
/sw.js (generated by VitePWA)
  ↓ (events)
'waiting' → needRefresh state → confirm dialog → skipWaiting → reload
```

**Pattern:** Following existing `useBackendStatus.ts` async state hook pattern for consistency with codebase conventions.

**Production Gating:** Service worker only registers when both conditions true:
- `'serviceWorker' in navigator` (browser support)
- `import.meta.env.PROD` (production build)
