# Anti-Pattern Registry — Project Global

> Entries here apply to ALL future phases in this project, not just one phase.
> Phase-local entries are in `.planning/phases/<phase>/.continue-here.md`.
> Future plan-phase / execute-phase orchestrators should read BOTH this file
> and the active phase's local `.continue-here.md`.

| Anti-pattern | Severity | Manifestation | Structural prevention |
|--------------|----------|---------------|------------------------|
| Writing files to root `server/`, `prisma/`, or `src/` instead of `apps/web/...` | blocking | Phase 36-02 wrote 3 webhook source files to root `server/` instead of `apps/web/server/`, leaving the live backend with a 404 on every Stripe event for ~2 days. Three milestone-36 incidents total (36.1 schema additions claimed but never written; 36.2 schema models claimed but never written; 36.3 webhook files written to wrong path). | Post-36.3, root `server/`, `prisma/`, `src/`, `tsup.config.ts`, `prisma.config.ts` are deleted — they cannot be a write target by accident. Future executor agents reading this file MUST verify their `<files_modified>` paths begin with `apps/web/`, `packages/`, or top-level config files (`package.json`, `pnpm-workspace.yaml`, `.github/`, etc.) before commit. |

## Compliance check (for plan-phase / execute-phase)

When scoping `<files_modified>` for any new plan, the orchestrator MUST verify each path against this allow-list of valid roots:

- `apps/web/...`        (live application — backend, frontend, schema)
- `apps/<other>/...`    (future apps in monorepo)
- `packages/...`        (shared workspace packages)
- `.github/...`         (CI / workflows)
- `.planning/...`       (planning artifacts)
- `pnpm-workspace.yaml`, `package.json`, `pnpm-lock.yaml`, `docker-compose.yml`, `Dockerfile`, `.gitignore` (top-level config)
- `tsconfig.base.json`  (only if it survived the 36.3 orphan sweep — check existence first)

Any path beginning with `server/`, `prisma/`, `src/` (at repo root) is FORBIDDEN. These are the wrong-path failure modes that caused the 36.x incidents. The orphan sweep in 36.3-03 deleted them; recreating them silently is a regression.
